## [0.3.0] - 2026-01-17

### Highlights

- Major architecture refactor: reorganized into modular subpackages by operation type
- Added comprehensive GitHub API operations for repositories, pull requests, and releases
- Maintained backward compatibility with v0.2.x API (deprecated, will be removed in v0.4.0)

### Added

- New `auth` package with `NewGitHubClient`, `GetAuthenticatedUser`, `GetUser` functions
- New `config` package with `Config` struct, `FromEnv`, `FromEnvWithConfig`, GitHub Enterprise support
- New `errors` package with standard error types (`ErrNotFound`, `ErrPermissionDenied`, `ErrRateLimited`, etc.) and `Translate` function
- New `pathutil` package with `Validate`, `Normalize`, `Join`, `Split`, `Dir`, `Base`, `Ext`, `HasPrefix` functions
- New `search` package with `Client`, `SearchIssues`, `SearchIssuesAll`, `Query` builder, and `Issues` type with table generation
- New `repo` package with fork operations (`EnsureFork`, `GetDefaultBranch`)
- New `repo` package with branch operations (`CreateBranch`, `GetBranchSHA`, `DeleteBranch`, `BranchExists`)
- New `repo` package with commit operations (`CreateCommit`, `ReadLocalFiles`, `FileContent` type)
- New `repo` package with listing operations (`ListOrgRepos`, `ListUserRepos`, `GetRepo`, `ParseRepoName`)
- New `repo` package with batch operations (`Batch` type for atomic multi-file commits)
- New `pr` package with `CreatePR`, `GetPR`, `ListPRs`, `MergePR`, `ClosePR`, `AddPRReviewers`, `ListPRFiles`, `ListPRComments`
- New `release` package with `ListReleases`, `ListReleasesSince`, `GetRelease`, `GetLatestRelease`, `GetReleaseByTag`, `ListReleaseAssets`
- New `cliutil` package with `GitStatusShortLines`, `GitRmDeletedLines`, `GitRmDeletedFile` helpers
- Example command `cmd/searchuserpr` for searching user pull requests

### Changed

- Reorganized codebase from flat package structure to modular subpackages by operation type
- Root package now provides backward-compatible re-exports delegating to subpackages
- Moved search functionality from `api_search.go` to `search/search.go`
- Moved constants from `constants.go` to `search/query.go`
- Moved issue types from `issue.go` to `search/issues.go`
- Moved example command from `pullrequest/cmd/searchuserpr` to `cmd/searchuserpr`

### Deprecated

- Root package `Client` type - use `auth.NewGitHubClient` and subpackage clients directly
- Root package `NewClient` function - use `auth.NewGitHubClient` instead
- Root package `NewClientWithToken` function - use `auth.NewGitHubClient` instead
- Root package type aliases (`Query`, `Issues`, `Issue`) - import from `search` package directly
- Root package constant re-exports (`ParamUser`, `ParamState`, etc.) - import from `search` package directly
- Root package wrapper methods (`SearchOpenPullRequests`, `SearchIssues`, `SearchIssuesAll`) - use `search.Client` methods directly

### Removed

- Removed `api_search.go` (functionality moved to `search/search.go`)
- Removed `constants.go` (functionality moved to `search/query.go`)
- Removed `issue.go` (functionality moved to `search/issues.go`)
- Removed `pullrequest/` directory (functionality moved to `pr/` and `cmd/`)

### Dependencies

- Upgraded `github.com/google/go-github` from v68 to v81
- Updated Go version from 1.22 to 1.24.0
- Updated indirect dependencies

### Documentation

- Updated README with new modular package structure and usage examples
- Added RELEASE_NOTES_v0.3.0.md with comprehensive release documentation
- Added ROADMAP.json and ROADMAP.md with project roadmap and v0.4.0 deprecation plan
